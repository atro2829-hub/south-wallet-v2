## 2. إدارة مزودي API

### 2.1 الوصول للوحة G2Bulk

من القائمة الجانبية: **"مزودي API"** → **"G2Bulk"**

تظهر لوحة بـ 4 تبويبات:

### 2.2 تبويب "الإعدادات"

![Settings Tab]

**الحقول:**
- **مفتاح G2Bulk API** — يُدخل يدوياً (موجود مسبقاً في النظام)
- **نسبة الربح (%)** — تُضاف لسعر التكلفة لتحديد سعر البيع

**الأزرار:**
- **"حفظ الإعدادات"** — يحفظ المفتاح ونسبة الربح في `api_providers` table
- **"اختبار الاتصال"** — يستدعي `GET /v1/getMe` ويعرض:
  - اسم المستخدم لدى G2Bulk
  - الرصيد الحالي بالدولار

### 2.3 تبويب "المزامنة"

![Sync Tab]

**الإحصائيات المعروضة:**
- عدد الفئات المُزامَنة
- عدد المنتجات المُزامَنة
- عدد الألعاب المُزامَنة
- تاريخ آخر مزامنة

**أزرار المزامنة:**

| الزر | الوظيفة | المدة المتوقعة |
|------|--------|---------------|
| **مزامنة الفئات** | يجلب 151 فئة من `/v1/category` ويُنشئ sub-sections تحت `g2bulk-root` | ~30 ثانية |
| **مزامنة المنتجات** | يجلب المنتجات من `/v1/products` ويُنشئ `service_providers` + `product_packages` | ~2 دقيقة |
| **مزامنة الألعاب** | يجلب 206 لعبة من `/v1/games` ويُخزِّنها في `api_games` | ~20 ثانية |
| **مزامنة شاملة** | يُشغّل الثلاث بالتوازي | ~2-3 دقائق |

**سجل المزامنة المباشر:**
في الأسفل يظهر سجل بكل عملية مزامنة مع الوقت والنتيجة (✓ نجح / ✗ فشل).

> 💡 **ملاحظة:** المزامنة الشاملة تجلب الأقسام والمنتجات والألعاب بالتوازي. الفئات تُنشأ كـ sub-sections تحت القسم الرئيسي `g2bulk-root`، والألعاب تُخزَّن في جدول `api_games` وتظهر تحت قسم "الألعاب" في تطبيق المستخدم.

### 2.4 تبويب "الألعاب"

![Games Tab]

**الوظيفة:** تصفح الألعاب المُزامَنة بصورها وأيقوناتها.

**الميزات:**
- **بحث فوري** بالاسم أو الكود
- **شبكة عرض** (grid) بـ 6 أعمدة على الشاشات الكبيرة
- كل لعبة تظهر بصورتها من G2Bulk CDN
- الضغط على لعبة → ينقل لتبويب "الاختبار" مع تحميل بيانات اللعبة

**مثال على الألعاب المعروضة:**
- PUBG Mobile (`pubgm`)
- Free Fire (`freefire`)
- Mobile Legends Bang Bang (`mlbb`)
- Call of Duty Mobile (`codm`)
- Genshin Impact (`genshin`)
- ... 206 لعبة إجمالاً

### 2.5 تبويب "الاختبار"

![Test Tab]

**الوظيفة:** تجربة تدفق شراء لعبة كامل قبل الإطلاق للمستخدمين.

**الخطوات:**

1. **اختر لعبة** (من تبويب "الألعاب" أو اكتب الكود يدوياً)
2. يعرض النظام:
   - صورة اللعبة واسمها وكودها
   - الحقول المطلوبة (مثلاً: `userid` أو `userid, serverid`)
3. **أدخل معرف اللاعب** (Player ID)
4. اضغط **"تحقق"**
   - يستدعي `POST /v1/games/checkPlayerId`
   - يعرض النتيجة:
     - ✅ ✓ الاسم: `PRIME・AKASH`
     - ❌ ✗ معرف غير صالح
5. (إن لزم) **أدخل معرف السيرفر** (Server ID)
6. **اختر باقة** من قائمة `GET /v1/games/{code}/catalogue`
   - كل باقة تظهر باسمها وسعرها ($0.88, $1.75, ...)
7. اضغط على باقة → **"تجربة شراء"**
   - يستدعي `POST /v1/games/{code}/order`
   - يعرض النتيجة:
     - ✅ Order ID, Status, Price
     - يتم polling تلقائي للحالة كل 5 ثوانٍ
     - يعرض عناصر التسليم (delivery_items) عند اكتمال الطلب

> ⚠️ **تنبيه:** زر "تجربة شراء" يُنفذ طلباً حقيقياً ويخصم من رصيد G2Bulk. استخدمه فقط للاختبار، واختر باقة رخيصة (مثل 60 UC).

## 3. إدارة الأقسام

### 3.1 القائمة الجانبية: "الأقسام"

**الوظيفة:** إضافة/تعديل/حذف الأقسام الرئيسية التي تظهر في تطبيق المستخدم.

**الحقول لكل قسم:**
- **المعرف (id)** — نصي، فريد (مثلاً: `telecom`, `games`, `g2bulk-root`)
- **الاسم بالعربية** و **الاسم بالإنجليزية**
- **الأيقونة** — emoji أو رابط صورة
- **اللون** — كود HEX (مثلاً: `#8B5CF6`)
- **النوع (type):**
  - `manual` — قسم عادي بمزودين يدويين
  - `api` — قسم يعرض منتجات من مزود API
  - `games` — قسم يعرض الألعاب (يفتح شاشة التحقق من ID)
  - `wallet` — قسم المحفظة (USDT، تحويلات)
  - `exchange` — صرافة عملات
  - `telecom` — شحن اتصالات
  - `investment` — استثمارات
  - `escrow` — ضمان
- **مزود API (api_provider_id)** — يربط القسم بمزود (مثلاً: `g2bulk`)
- **نوع قسم API (api_section_type):** `products` / `games` / `categories`
- **عرض في الرئيسية (show_in_home)** — BOOL
- **عرض في الخدمات (show_in_services)** — BOOL
- **الترتيب (sort_order)** — رقم

### 3.2 الأقسام الافتراضية بعد الإصلاح

```
sort_order=100  telecom          "الاتصالات"
sort_order=200  entertainment    "ترفيه"
sort_order=300  gift-cards       "بطاقات هدايا"
sort_order=400  digital-wallets  "محافظ رقمية"
sort_order=800  g2bulk-root      "G2Bulk"        ← قسم رئيسي للمنتجات
sort_order=900  g2bulk-games     "الألعاب"        ← قسم رئيسي للألعاب
```

## 5. إدارة الطلبات

### 5.1 القائمة الجانبية: "الطلبات"

**الوظيفة:** عرض كل الطلبات، تصفيتها، إكمالها يدوياً، إلغاؤها.

**حالات الطلب:**
- `pending` — قيد الانتظار
- `processing` — قيد المعالجة
- `completed` — مكتمل
- `failed` — فشل
- `refunded` — تم استرداد المبلغ
- `cancelled` — ملغى

### 5.2 طلبات G2Bulk المحددة

للعثور على طلبات G2Bulk:
1. افتح تبويب "الطلبات"
2. فلتر بـ `api_provider_id = g2bulk`
3. ستظهر طلبات الألعاب مع:
   - `game_code` — اللعبة (مثلاً: `pubgm`)
   - `customer_input` — معرف اللاعب
   - `player_name` — اسم اللاعب المُتحقَّق منه (جديد!)
   - `player_id_verified` — تم التحقق؟ (جديد!)
   - `api_order_id` — رقم الطلب لدى G2Bulk
   - `g2bulk_order_status` — آخر حالة polling (جديد!)
   - `api_response` — الرد الكامل من G2Bulk (JSONB)

### 5.3 استرداد رصيد يدوي

في حال فشل الاسترداد التلقائي (نادر)، يمكن استرداد الرصيد يدوياً:

1. افتح الطلب في تبويب "الطلبات"
2. اضغط "استرداد الرصيد"
3. النظام سيضيف المبلغ لرصيد المستخدم ويحدّث الطلب إلى `refunded`

أو عبر SQL مباشرة:
```sql
-- 1. استرداد الرصيد
UPDATE public.users
SET balance_usd = balance_usd + 0.99  -- المبلغ
WHERE id = 'USER_UUID';

-- 2. تحديث الطلب
UPDATE public.orders
SET status = 'refunded',
    g2bulk_order_status = 'FAILED',
    updated_at = NOW()
WHERE id = 'ORDER_UUID';
```

## 8. استكشاف الأخطاء

### 8.1 "لا يوجد مزود API مُكوّن"

**السبب:** قديماً كان يعني أن `adminSettings/apiProviders` فارغ. **تم الإصلاح** — الآن يقرأ من `api_providers` table في Supabase.

**الحل الحالي:** تأكد من وجود صف في `api_providers` بالـ id = `g2bulk`:
```sql
SELECT id, name, is_active, api_key FROM api_providers WHERE id = 'g2bulk';
```

### 8.2 "رصيد المزود غير كافٍ"

**السبب:** رصيد G2Bulk نفد.

**الحل:** سجّل دخول في https://g2bulk.com وأضف رصيداً.

### 8.3 "فشل المزامنة: HTTP 429"

**السبب:** تم تجاوز حد الطلبات (rate limit).

**الحل:** انتظر دقيقة وأعد المزامنة.

### 8.4 الطلبات تبقى في PENDING

**السبب الطبيعي:** G2Bulk يستغرق وقتاً للمعالجة (عادةً 1-5 دقائق).

**التحقق:**
```sql
SELECT id, game_code, customer_input, api_order_id, g2bulk_order_status, created_at
FROM orders
WHERE api_provider_id = 'g2bulk'
  AND status = 'pending'
ORDER BY created_at DESC;
```

### 8.5 صور الألعاب لا تظهر

**السبب:** إما أن G2Bulk CDN محجوب في شبكة المستخدم، أو أن `image_url` فارغ.

**الحل:** تأكد من أن `image_url` يبدأ بـ `https://api.g2bulk.com/images/...`:
```sql
SELECT game_code, name, image_url FROM api_games
WHERE image_url IS NULL OR image_url = '' OR image_url NOT LIKE 'https://%';
```
