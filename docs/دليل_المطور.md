### 3.3 الـ RLS Patterns

**النمط الصحيح (بدون recursion):**

```sql
-- 1. أنشئ دالة SECURITY DEFINER
CREATE FUNCTION public.is_admin()
RETURNS BOOLEAN
LANGUAGE sql SECURITY DEFINER STABLE
SET search_path = public
AS $$
  SELECT EXISTS (
    SELECT 1 FROM public.users
    WHERE id = auth.uid() AND role IN ('admin', 'owner', 'super_admin')
  );
$$;

-- 2. استخدمها في السياسات
CREATE POLICY "Admin full access on orders"
  ON public.orders FOR ALL
  USING (public.is_admin())
  WITH CHECK (public.is_admin());
```

**النمط الخاطئ (recursion):**

```sql
-- ❌ هذا يسبب "infinite recursion detected in policy"
CREATE POLICY "..." ON public.users FOR ALL
  USING (
    EXISTS (SELECT 1 FROM users WHERE id = auth.uid() AND role = 'admin')
  );
```

### 3.4 PostgREST Upsert

عند عمل upsert عبر REST API، الـ `On Conflict` header يجب أن يطابق قيد فريد (UNIQUE constraint) موجود:

```http
POST /rest/v1/api_categories
Prefer: resolution=merge-duplicates
On Conflict: api_provider_id,api_category_id
Content-Type: application/json

[
  {"api_provider_id": "g2bulk", "api_category_id": "1", ...},
  {"api_provider_id": "g2bulk", "api_category_id": "2", ...}
]
```

> إذا ظهر خطأ `duplicate key value violates unique constraint`، تحقق أن:
> 1. القيد الفريد موجود (وليس مجرد index)
> 2. الـ `On Conflict` header يطابق أعمدة القيد بالضبط
> 3. لا يوجد قيود فريدة متعددة على نفس الأعمدة ( migration 020 يُصلح هذا)

## 4. تكامل G2Bulk — شرح معمَّق

### 4.1 الـ Edge Function Proxy

```typescript
// supabase/functions/g2bulk-proxy/index.ts
Deno.serve(async (req) => {
  // 1. تحقق من auth (apikey أو JWT)
  const apiKey = req.headers.get('apikey');
  if (!apiKey) return new Response('Missing auth', { status: 401 });

  // 2. اجلب مفتاح G2Bulk من env secret
  const g2bulkKey = Deno.env.get('G2BULK_API_KEY');
  if (!g2bulkKey) return new Response('Not configured', { status: 500 });

  // 3. استخرج الـ path بعد /g2bulk-proxy
  const url = new URL(req.url);
  const path = url.pathname.replace(/^\/functions\/v1\/g2bulk-proxy/, '');

  // 4. مرر الطلب إلى G2Bulk
  const targetUrl = `https://api.g2bulk.com${path}${url.search}`;
  const upstream = await fetch(targetUrl, {
    method: req.method,
    headers: {
      'X-API-Key': g2bulkKey,  // ← المفتاح يُضاف هنا فقط
      'Content-Type': 'application/json',
    },
    body: req.method !== 'GET' ? await req.text() : undefined,
  });

  // 5. أعد الرد
  return new Response(await upstream.text(), {
    status: upstream.status,
    headers: { 'Content-Type': 'application/json', ...corsHeaders },
  });
});
```

### 4.2 التوجيه في `api-providers.ts`

```typescript
async function apiRequest<T>(provider, endpoint, method, body) {
  const isG2Bulk = provider.type === 'g2bulk' || provider.baseUrl.includes('g2bulk.com');

  if (isG2Bulk) {
    // ← مرّر عبر الـ proxy
    url = `${G2BULK_PROXY_URL}${endpoint}`;
    headers = { 'apikey': SUPABASE_ANON_KEY };
  } else {
    // ← اتصال مباشر (للمزودين الآخرين)
    url = `${provider.baseUrl}${endpoint}`;
    headers = { [provider.authHeaderName]: provider.apiKey };
  }

  const response = await fetch(url, { method, headers, body: JSON.stringify(body) });
  // ... error handling ...
}
```

## 8. استكشاف الأخطاء (Development)

### 8.1 "infinite recursion detected in policy for relation"

**السبب:** سياسة RLS تستعلم عن نفس الجدول الذي تحميه.

**الحل:** استخدم دالة `SECURITY DEFINER` (انظر §3.3).

### 8.2 "permission denied for table X"

**السبب:** GRANTs ناقصة على الدور المتصل (anon أو authenticated).

**الحل:** شغّل migration 018 أو:
```sql
GRANT SELECT, INSERT, UPDATE, DELETE ON public.X TO anon, authenticated;
```

### 8.3 "column Y does not exist"

**السبب:** migration لم يُطبَّق بالكامل.

**الحل:** تحقق من الأعمدة الفعلية:
```sql
SELECT column_name FROM information_schema.columns 
WHERE table_schema='public' AND table_name='X';
```

### 8.4 "Could not find the 'Y' column of 'X' in the schema cache"

**السبب:** PostgREST لم يُحدّث cache بعد إضافة عمود.

**الحل:**
```sql
NOTIFY pgrst, 'reload schema';
```

### 8.5 Cloudflare HTTP 403 على Supabase Management API

**السبب:** WAF يحظر الطلبات بدون User-Agent مناسب.

**الحل:** أضف `User-Agent` header:
```python
headers["User-Agent"] = "south-wallet-migrator/1.0"
```

### 8.6 Edge Function BOOT_ERROR

**السبب المحتمل:** استخدام `import { serve }` القديم بدلاً من `Deno.serve`.

**الحل:** استخدم `Deno.serve()` المدمج:
```typescript
// ❌ قديم — قد يفشل
import { serve } from 'https://deno.land/std@0.168.0/http/server.ts';
serve(async (req) => { ... });

// ✅ جديد — مُوصى به
Deno.serve(async (req) => { ... });
```

| `product_packages` | باقات الشراء | قراءة عامة، كتابة للأدمن |
| `api_providers` | مزودو API الخارجيون | قراءة عامة، كتابة للأدمن |
| `api_games` | الألعاب المُزامَنة | قراءة عامة، كتابة للأدمن |
| `api_categories` | فئات من API | قراءة عامة، كتابة للأدمن |
| `api_products` | منتجات من API | قراءة عامة، كتابة للأدمن |
| `api_game_catalogues` | باقات شحن الألعاب | قراءة عامة، كتابة للأدمن |
| `orders` | الطلبات | المستخدم يرى طلباته، الأدمن يرى الكل |
| `transactions` | المعاملات المالية | المستخدم يرى معاملاته، الأدمن يرى الكل |

### 9.3 RPCs (PostgreSQL Functions)

```sql
-- تحديث رصيد المستخدم ذرياً
SELECT update_user_balance(p_user_id, p_currency, p_amount, p_operation);
-- p_operation: 'add' | 'subtract' | 'set'

-- تحويل ذري بين مستخدمين
SELECT transfer_money(p_from_user_id, p_to_user_id, p_amount, p_currency, p_description);

-- معالجة طلب ذرياً (خصم + إنشاء transaction + إنشاء order)
SELECT process_order(p_user_id, p_provider_id, ...);

-- إحصائيات لوحة الأدمن
SELECT get_dashboard_stats();
SELECT get_provider_stats();
```
